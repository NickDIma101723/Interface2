verwijderen. En natuurlijk moet je de afspraken overzichtelijk op het scherm zien.

Read, Update en Delete). De volledige applicatie wordt gemaakt in PHP. Als je niet meer weet
hoe je ook alweer een PHP-applicatie maakt, kijk dan nog eens naar de readers uit leerjaar 1.

SQL

Om gegevens in een database te kunnen benaderen (toevoegen, uitlezen, aanpassen of
verwijderen), gebruiken we een speciale taal: SQL (Structured Query Language). Met deze taal
kan je via queries (vragen/opdrachten) de database benaderen.
Deze taal heeft verschillende ‘dialecten’, oftewel versies. Zoals PostgreSQL, SQLite en MySQL.
En dan heb je ook nog NoSQL-databases….
PHP gebruikt MySQL om databasegegevens te benaderen.

Neem de volgende tutorials door om meer te leren over SQL en MySQL:
   • https://icttrainingen.nl/blog/wat-is-sql
   • https://programmeerplaats.nl/wat-is-mysql/



Een database die via MySQL te benaderen is, noemen we een MySQL-database.
Jullie hebben de mogelijkheid om via PLESK een MySQL-database te maken. Daarvoor gebruiken
we een database management system (DBMS): PHPMyAdmin.

QUERIES

Op LinkedIn wordt de basis van MySQL en queries uitgelegd:
https://www.linkedin.com/learning/php-with-mysql-essential-training-1-the-basics/mysql-

In een tabel kunnen meerdere acties uitgevoerd worden. We gebruiken een query om aan te
geven wat er moet gebeuren.

De acties kunnen we verdelen in vier handelingen:

 Handeling                    Omschrijving                               Query
 Toevoegen       Create       Toevoegen van gegevens aan een tabel       “INSERT INTO tabelnaam …”
 Lezen           Read         Lezen van gegevens uit 1 of meerdere       “SELECT * FROM tabelnaam…”
                              tabellen
 Aanpassen       Update       Gegevens in een tabel aanpassen            “UPDATE tabelnaam SET …”
 Verwijderen     Delete       Verwijderen van gegevens uit een tabel     “DELETE FROM tabelnaam…”

De 4 handelingen zijn dus, in het Engels: Create, Read, Update en Delete. Afgekort: CRUD.
Als we in PHP een applicatie bouwen die al deze handelingen kan uitvoeren, noemen we dit een
CRUD-applicatie.

Hoe zien die queries er nu precies uit?
Dat gaan we eerst bekijken in PHPMyAdmin.

CONNECTION SCRIPT


Het eerste dat we nodig hebben is een verbinding vanuit PHP naar de database.

We maken net als in Leerjaar 1 gebruik van PDO, waarom je kiest voor MySQLi of PDO kan je hier
vinden, echter omdat we ook vorig jaar gebruik hebben gemaakt van PDO zetten we dat liefst
voort. Eerder werkte we met een SQLite (lokale) database file. De connectie die we daarvoor
maakte werkt niet voor MySQL. Om deze te laten werken moeten we de connectie aanpassen.


Meer informatie over de verschillende manieren van verbinden kan je hier vinden:
https://www.w3schools.com/php/php_mysql_connect.asp

We gebruiken NIET de MySQLi varianten, maar die van PDO!

Je ziet dat er een stukje PHP-code nodig is om een verbinding te maken met de database.
Daarin worden 4 eigenschappen voor de verbinding vastgelegd:


     •   $db_hostname = De naam van de server waar de database staat.
     •   $db_username = De naam van de eigenaar van de database (jouw gebruikersnaam).
     •   $db_password = Het wachtwoord van de eigenaar.
     •   $db_database = De naam van de database.


Daarna wordt er een mysqli-functie uitgevoerd om de verbinding te maken.
Deze verbinding wordt ‘bewaard’ in de variabele $mysqli. (Dit is eigenlijk een object.)

CONNECTION SCRIPT: ERRORS TONEN

De webserver van school is zo ingesteld dat PHP-fouten niet worden getoond. Dit is gedaan
vanwege de veiligheid: als een gebruiker fouten in de code kan lezen, dan zou hij/zij hier misbruik
van kunnen maken.
Als developer is het juist fijn als die fouten wél getoond worden. Gelukkig kan dat door een aantal
regels code toe te voegen; dit doen we in het bestand ‘config.php’. Omdat dit bestand in (bijna)
alle PHP-pagina’s wordt bijgevoegd (included) kunnen we zo op al deze pagina’s de fouten tonen.
LET OP: als een webapplicatie klaar is en opgeleverd wordt aan de klant, dan moet je deze regels
(voor de veiligheid) weer verwijderen uit de code!

Er zijn meerder opties om foutmeldingen te tonen. Zoek maar eens op Internet naar “PHP error
reporting”. Wij gebruiken de volgende regels:

       ini_set('display_errors', 1);
       error_reporting(E_ALL);

REGULIERE EXPRESSIES

Soms is er geen speciale functie om te controleren of de waarde voldoet aan bepaalde eisen.
Denk aan een postcode; die moet bestaan uit precies 4 letters en daarna 2 cijfers.
Om te controleren of de waarde aan een bepaald patroon voldoet, gebruik je reguliere expressies,
oftewel RegEx.
Hieronder een paar voorbeelden:

   •   RegEx voor het emailadres van een GLR-student: [0-9]{6}@glr\.nl
       Dit betekent:

           o Een getal tussen 0 en 9
           o Dat 5 keer (dus vijf getallen)
           o Gevolgd door de letterlijke tekst “@glr.nl” (de backslash voor de punt betekent dat
             de punt als letterlijk leesteken gezien moet worden, anders heeft de punt een
             functie in de “taal” RegEx).

   •   RegEx voor een postcode:         [1-9][0-9]{3} [A-Z]{2}
       Dit betekent:

           o   Een getal van 1 tot en met 9 (dus niet 0)
           o   Daarna drie getallen waarvan (0 t/m 9)
           o   Gevolgd door een spatie
           o   Gevolgd voor twee hoofdletters

LES 4 – GEGEVENS AANPASSEN

Het laatste onderdeel van de CRUD-applicatie dat nog behandeld moet worden is de Update,
oftewel het aanpassen van gegevens

Ook deze query heb je al kunnen zien als je in PHPMyADdmin een record aanpast:




De Update-query bestaat uit 3 onderdelen:
• Achter ‘UPDATE’ staat de tabel die aangepast moet worden.
• Achter ‘SET’ staan de kolommen en de nieuwe waarden; er kunnen meerdere kolommen in
    één keer worden aangepast.
• Achter WHERE staat de selectie: welke record(s) moeten worden aangepast? Als er maar één
    record moet worden aangepast, dan staat hier de sleutel van de tabel (‘ID’).

De query die we in PHP gebruiken ziet er iets anders uit:

       UPDATE crud_agenda SET Inhoud = 'Hoofdstuk 7 afmaken' WHERE ID = 3;

De meeste aanhalingstekens zijn weg, behalve (!) bij de tekst die aangepast moet worden.
En bij WHERE is de tabelnaam verwijderd. Deze hoeft niet genoemd te worden.

Bijna alle applicaties waarin met gegevens wordt omgegaan vereisen een vorm van beveiliging. Je

INLOG MET DATABASE

Het is veiliger om de inloggegevens te bewaren in een database, en niet in de php-pagina zelf.
Daarom ga je nu eerst een tabel maken met daarin de inloggegevens van de users.
Deze tabel (‘users’) zou er als volgt uit kunnen zien:




Een paar opmerkingen over de tabel:

   •   Het id is de key van de tabel en is autonummering.
       Als je zeker weet dat er geen users zullen zijn met dezelfde username, dan kan je het veld
       ‘username’ als key gebruiken.
   •   Het level is extra. Mocht je later verschil willen maken in userlevels, dan kan je deze
       gebruiken.
   •   Het wachtwoord wordt gehashed opgeslagen, zodat het niet leesbaar is voor anderen.
       In deze tabel is gekozen voor een sha1-beveiliging. sha1 is een string van 40 karakters en

       Dit is niet de veiligste methode maar wel eenvoudig. Zorg ervoor dat alle karakters van het
       wachtwoord in het veld passen; kies daarom bijvoorbeeld “varchar(100)”
